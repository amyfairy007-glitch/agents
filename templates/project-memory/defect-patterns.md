## Usage Notes

- Check the Quick Index first when working on a similar module or flow.
- Put the most reusable signal in `Key Signal`, not a long summary.
- Keep entries practical and short enough to scan quickly.
- Update an existing pattern when it is the same failure mode; create a new one only when the pattern is meaningfully different.
