## 二、对齐到目标效果图的具体项

- 新增最左侧 App Rail，作为独立一级导航。
- 将主工作区收敛为 App Rail + 项目栏 + Task 栏 + 右侧工作区的四列结构。
- 默认保持项目栏、Task 栏展开，不默认压缩成图标条。
- 右侧顶部改为紧凑上下文条，不再铺开多个大“暂无”卡片。
- 统一了深色蓝黑背景、低对比边框、蓝紫主强调色和文字层级。
- 项目栏、Task 栏和右侧工作区的正文、空状态、路径、时间、筛选文字全部提升为可读级别。
- 右侧通过“更多操作”提供纯前端示例预览，覆盖工作区示例、执行完成示例、产物示例和审批示例。

## 四、App Rail 实现

- 入口包括：项目、能力、Task / 工作流、产物、文档、设置。
- 当前阶段真实可用入口仅“项目”。
- 其他入口点击后只显示“当前阶段暂未开放。”，不会创建假数据或进入假页面。
- 当前入口使用背景和边框高亮，不只依赖颜色。

## 五、项目栏与 Task 栏可读性修复点

- 项目名称、路径、筛选文案和空状态说明改为更高亮度的文字色 Token。
- `项目详情` 按钮从说明文字中拆开，放在独立操作区，避免窄栏内挤成碎片。
- 项目栏增加稳定的“展开项目栏”按钮，折叠态始终可点击。
- Task 栏保留 `+ 新建任务`、状态筛选和当前选中项，空状态提供明确的主按钮入口。
- 折叠 / 展开按钮不再依赖圆点或模糊图标。

## 七、四种状态分别如何实现

### 1. 无 Task 状态

- 右侧只显示一个引导面板。
- 文案明确提示“还没有任务，或者尚未选择任务。”
- 主按钮只有一个：`+ 新建任务`。
- 不显示六个大“暂无”卡片，不显示假 Agent 输出、假产物、假审批壳。

### 2. 已选真实 Task、未绑定 Skill / SOP

- 右侧工作区采用三列结构：
  - 原始想法 / 任务说明
  - Skill / SOP 选择区
  - 当前步骤与下一步动作
- 明确显示“Capability Registry 尚未初始化”。
- 不写入真实绑定，不生成真实 SOP。

### 3. 纯前端示例预览

- 通过 `更多操作` 打开的菜单进入。
- 支持“预览工作区示例”。
- 页面会明确显示“示例预览，不会写入真实 Task 数据。”
- 仅用于视觉验收，不会写入 `data/ai-coding-console/`。

### 4. 执行完成示例预览

- 同样通过 `更多操作` 进入。
- 在 Agent 输出、产物、审批记录中展示纯前端示例内容。
- 明确标注“示例预览，不代表真实 Agent 执行结果”。
- 不会写入真实 Run、Artifact 或审批数据。

## 八、真实状态与示例状态边界

- 真实状态只依赖当前项目的真实 Task 数据、真实项目详情和现有 API。
- 示例状态只由前端 `workspacePreviewMode` 控制，刷新后不保留。
- 示例预览不会调用新的数据写入逻辑，也不会改写真实 Task。
- 这次只新增视觉层和前端状态切换，没有接入 Capability Registry、Prompt Builder、SOP 生成或 Agent 执行。
