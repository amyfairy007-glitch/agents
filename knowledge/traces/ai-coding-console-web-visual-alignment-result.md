# AI Coding Console 当前 Web 视觉对齐优化结果

> 生成日期：2026-07-05
> 范围：仅调整现有三栏 Web 工作台的视觉层与展示层，不改数据模型、不改 API 语义、不改 Task 流程
> 提交：`5075970b8cba7de6a34f54ef304f91124bb3e61c`

## 一、实际修改文件

- `tools/ai-coding-console/gui/index.html`
- `tools/ai-coding-console/gui/app.js`

## 二、对齐到目标效果图的具体项目

- 默认三栏比例收紧为更接近目标图的高密度工作台：项目栏约 248px，Task 栏约 320px，右侧工作区占剩余空间。
- 顶部上下文由原先较分散的状态卡，压缩成更紧凑的单条上下文栏。
- 左栏项目列表保留高亮、搜索和项目详情入口，视觉上更像项目切换侧栏而不是后台菜单。
- 中栏 Task 列表保留 `+ 新建任务`、状态筛选和当前选中项高亮，但整体密度更高、留白更少。
- 右侧无 Task 状态改为单主动作启动引导，不再平铺 Prompt / Agent / Artifact / 审批的空壳内容。
- 项目详情抽屉保留结构化 Git / AGENTS / `.ai` 展示，但不占据首页主视觉。
- 深色蓝黑背景、低对比边框、统一圆角和按钮密度更接近目标参考图。

## 三、未能完全对齐的部分及原因

- 当前页面仍保留既有工作台的部分信息密度和过渡效果，没有完全重做成目标图里更偏“产品化”的极简视觉系统。
- 由于必须保留现有 Task / Prompt / SOP / Artifact 结构，右侧在有 Task 时仍会比目标图更像“功能完备的工作台”，而不是纯视觉展示页。
- 项目详情抽屉仍保留现有结构化字段，尚未做进一步的内容精简或图标化收敛。

## 四、默认布局尺寸

- 项目栏：`248px`
- Task 栏：`320px`
- 右侧工作区：`minmax(0, 1fr)`
- 顶部上下文栏：紧凑单行标题 + pill 结构

## 五、折叠 / 展开行为

- 项目栏默认展开，顶部保留明确的折叠按钮。
- Task 栏默认展开，顶部保留稳定的折叠按钮。
- 折叠后仍保留清晰可点击入口，不依赖圆点或模糊图标恢复。
- 刷新页面后，折叠 / 展开入口仍可稳定使用。

## 六、无 Task 状态

- 无 Task 时，右侧显示启动引导：
  - 当前项目名称
  - 还没有任务，或者尚未选择任务
  - `[+ 新建任务]`
  - 示例引导语句
- 右侧不再显示 Prompt / Agent / Artifact / 审批的空壳占位。

## 七、有 Task 状态

- 创建临时 Task 后，中栏能正确显示新任务。
- 点击任务后，右侧进入任务工作区。
- 顶部上下文显示项目、任务、状态、能力、Agent 和 Git 摘要。
- 工作区、Prompt 与 SOP、Agent 输出、产物、审批 Tab 均可继续切换。

## 八、浏览器实际验证结果

- `npm run gui` 对应的本地 GUI 服务可用。
- 默认项目栏和 Task 栏均展开。
- 项目栏可稳定折叠并恢复。
- Task 栏可稳定折叠并恢复。
- 当前项目切换后，中栏与右栏同步刷新。
- 无 Task 状态下，右侧显示新的启动引导。
- 通过 GUI 创建了临时 Task，Task 正确进入中栏。
- 点击临时 Task 后，右侧进入对应工作区。
- 项目详情抽屉可正常打开，展示结构化 Git / AGENTS / `.ai` 信息。
- Tab、项目详情抽屉、现有 Task 创建能力均保持可用。
