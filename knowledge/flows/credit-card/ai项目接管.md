# 6. 页面初始化、校验、Submit、请求、异常处理流程

这是重点。

请对比 AI 项目和成熟仓库中同类页面的完整流程：

```text
进入页面
→ 初始化
→ 加载数据
→ 用户输入
→ 点击 Search / Submit / Save / Delete
→ 触发校验
→ 错误回填
→ 校验失败中断
→ 参数组装
→ API 调用
→ loading
→ 成功处理
→ 异常处理
→ 刷新 / 跳转 / 返回
```

对每个流程节点都要判断：

* AI 项目当前实现在哪里；
* 成熟库标准流程在哪里；
* 当前是否重复手写流程；
* 当前是否绕过公共能力；
* 当前业务内容应保留在哪里；
* 当前流程代码应迁移、替换、适配还是保留；
* 是否受依赖版本影响；
* 改完后需要验证什么。

必须明确区分：

## 当前 AI 项目必须保留的内容

例如：

* 字段；
* 校验规则内容；
* 条件校验；
* 接口标识；
* 请求字段映射；
* 返回字段映射；
* 成功后的业务动作；
* 失败后的业务动作；
* 页面业务状态。

## 当前 AI 项目需要向成熟库对齐的内容

例如：

* 校验文件组织；
* 校验触发链路；
* 错误回显方式；
* Submit 调用顺序；
* 请求参数组织位置；
* API 调用入口；
* loading / toast / error 机制；
* 页面刷新、跳转、返回的技术实现方式。

# 8. 差距分级

所有差距必须标记优先级：

## P0：阻碍框架对齐或存在明显运行风险

例如：

* 页面绕过项目入口；
* 手写请求导致无法统一处理异常；
* Submit 流程无法验证；
* 组件库完全不兼容；
* 校验链路断裂；
* 关键页面没有统一错误处理。

## P1：核心流程不一致

例如：

* 校验、Submit、请求、初始化、组件接入方式不一致；
* 文件职责明显混乱；
* 多个页面重复实现同一流程。

## P2：结构和维护性问题

例如：

* 常量、文案、工具函数放置不一致；
* 页面过重；
* 重复组件；
* 文件命名不一致。

## P3：后续治理项

例如：

* 无效代码；
* TODO；
* 历史遗留；
* 可优化但暂不影响框架对齐的内容。
